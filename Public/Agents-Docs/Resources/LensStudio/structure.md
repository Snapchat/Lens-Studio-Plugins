# Lens Studio Project Structure

## Directory Layout

```
ProjectName/
├── Project.esproj          # Main project file (YAML format)
├── Assets/                 # All project assets (work here)
├── Cache/                  # Auto-generated files (DO NOT modify)
├── Support/               # TypeScript definitions (gitignored)
├── PluginsUserPreferences/ # Plugin settings (gitignored)
├── Workspaces/            # Editor workspace state (gitignored)
├── BackUp/                # Project backups (gitignored)
├── .gitignore             # Version control exclusions
├── .gitattributes         # Git LFS and merge settings
└── AGENTS.md              # Agent context file (this system)
```

## Project File (Project.esproj)

The main project file is YAML-based and contains:
- Scene hierarchy definitions
- Asset references and import settings
- Component configurations
- Build settings

You shouldn't modify it directly, but use tools to modify the state instead.

## Assets Folder

> **Note:** The `Assets/` directory is the only directory where you should create or edit files. All project resources — scripts, textures, materials, meshes, audio, and prefabs — belong here.

The assets folder has no strict rules for organization. This is one option to keep things organized.

### Scripts/

Contains all script files:

```
Scripts/
├── Components/           # Reusable TypeScript components
│   ├── RotateObject.ts
│   └── TapHandler.ts
├── Managers/             # Singleton/manager scripts
│   └── GameManager.ts
```

**File extensions:**
- `.ts` - TypeScript files (recommended)
- `.js` - JavaScript files

### Textures/

Supported formats:
- PNG, JPG, GIF, PVR, TIFF, DDS, TGA, BMP - Standard images
- GIF - Animated textures (converted to sprite sheets)
- MP4 (H.264 / AVC) - Video

### Materials/

Material files are YAML-based (`.material` extension) and reference:
- Shader graphs or passes
- Texture assignments
- Material properties

Create and modify them with your tools, not directly.

### Meshes/

Supported 3D formats:
- FBX, OBJ, GLTF/GLB

### Audio/

Supported formats:
- MP3, (Preferred) WAV

### Prefabs/

Prefabs (`.prefab` extension) are reusable scene object hierarchies:
- Store complete object configurations
- Can be instantiated at runtime
- Changes to prefabs update all instances

Create and modify them with your tools, not directly.

## Scene Hierarchy

The scene is organized as a tree of SceneObjects:

```
Scene (root)
├── Camera
│   └── [Camera Component]
├── Lighting
│   ├── DirectionalLight
│   └── AmbientLight
├── UI
│   ├── Canvas
│   │   ├── Title [Text]
│   │   └── Button [Image]
│   └── SafeArea
├── 3D Objects
│   ├── Character [RenderMeshVisual]
│   └── Environment [RenderMeshVisual]
└── Audio
    └── BackgroundMusic [AudioComponent]
```

## Component System

SceneObjects contain Components that provide functionality:

### Common Component Types

| Component | Purpose |
|-----------|---------|
| `Transform` | Position, rotation, scale (always present) |
| `ScriptComponent` | Custom script logic |
| `Camera` | Renders the scene |
| `Text` | 2D text display |
| `Image` | 2D image display |
| `RenderMeshVisual` | 3D mesh rendering |
| `AudioComponent` | Sound playback |
| `Animation` | Skeletal animation |
| `Physics.BodyComponent` | Physics simulation |
| `Physics.ColliderComponent` | Collision detection |

SceneObjects can have more than one Component.

### Script Components

Scripts attach to SceneObjects as components:

```typescript
@component
export class MyScript extends BaseScriptComponent {
    // Script is a component on a SceneObject
    onAwake() {
        const myObject = this.getSceneObject();
        const myTransform = this.getTransform();
    }
}
```

## Layers and Render Order

Objects can be assigned to layers for:
- Selective camera rendering
- Physics collision filtering
- Raycast filtering

```typescript
// Assign layer
sceneObj.layer = LayerSet.fromNumber(1);

// Camera renders specific layers
camera.renderLayer = LayerSet.fromNumber(1);
```

## Asset References

Assets are referenced by path or UUID:

```typescript
@component
export class AssetExample extends BaseScriptComponent {
    // Reference via Inspector (required - will fail if not assigned)
    @input
    myTexture: Texture;

    @input
    myMaterial: Material;

    // Optional references (use @allowUndefined)
    @input
    @allowUndefined
    myAudio: AudioTrackAsset;

    @input
    @allowUndefined
    myMesh: RenderMesh;
}
```
## Build Outputs

When building a Lens:
- Scripts are compiled and bundled
- Assets are optimized and compressed
- A `.lns` file is generated for upload

## Version Control Notes

> **Warning:** Never manually edit or commit files in `Cache/`. This directory is entirely auto-generated by Lens Studio and is regenerated every time the project loads.

Files automatically managed by Lens Studio:
- `Cache/` - Regenerated on load
- `Support/` - Contains generated `.d.ts` files
- `PluginsUserPreferences/` - User-specific settings
- `Workspaces/` - Editor state
- `BackUp/` - Automatic backups
- `.agents/LensStudio/` - Documentation for agents

These directories have guard `.gitignore` files to prevent accidental commits.

## Best Practices

1. **Organize by feature** - Group related assets together
2. **Use Prefabs** - For reusable object hierarchies
3. **Name consistently** - Use clear, descriptive names
4. **Keep Scripts separate** - Don't mix with other asset types
5. **Use TypeScript** - Better tooling and type safety
6. **Reference via @input** - Avoid hardcoded asset paths
7. **Use imports** - Share code between TypeScript files
